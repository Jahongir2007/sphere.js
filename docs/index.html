Why Sphere.js

Sphere.js is used to create simple
http servers. This framework is a good choice
for beginners. Learning Sphere.js is very quick and very easy.

const sphere = require("./sphere");
const app = sphere;
app.options({
    head: 200,
    send: "Creating HTTP server",
    listen: 8000
});

Getting started

gh repo clone Jahongir2007/sphere.js Download the framework using the GitHub CLI. Creating helloworld.js program: After downloading the framework, enter these codes:

const sphere = require("./bin/sphere");
const app = sphere;
app.options({
    head: 200,
    send: "Hello, world!",
    listen: 8000
});
Now we use it: 
node helloworld.js

Importing Sphere.js and creating sphere.js application

To import Sphere.js, you will need the require() method. 

const require = require("./sphere"); // importing Sphere.js
We save the Sphere.js methods to the desired variable (and our first program is ready): 
const app = sphere;

options() method in Sphere.js

Now let's use the options() method: 

const sphere = require("./sphere");
const app = sphere;
app.options(); // calling method options()
This method for creating http servers. Objects in the options() method 
const sphere = require("./sphere");
const app = sphere;
app.options({
    head: 200
});
The head: (Added in v1..0) property is an inbuilt property of the ‘http’ module which sends a response header to the request. The status code is a 3-digit HTTP status code, like 404. The last argument, headers, are the response headers. Optionally one can give a human-readable statusMessage as the second argument.
send: object for getting datas in the http server: 
const sphere = require("./sphere");
const app = sphere;
app.options({
        send: "Hello, world!" // getting Hello, world!
});
The listen: object for indicates on which server the data will be output. This object must be in the all methods: 
const sphere = require("./sphere");
const app = sphere;
app.options({
    listen: 8000
});

html() method in Sphere.js

const sphere = require("./sphere");
const app = sphere;
app.html({
    file: "/test.html",
    listen: 8000
    });
file: object indicates which file to retrieve. 
const sphere = require("./sphere");
const app = sphere;
app.html({
    file: "/test.html", // test.html
    listen: 8000
});

post() method in Sphere.js

post() method creating for POST: 

const sphere = require("./sphere");
const app = sphere;
app.post({
    send: "Sphere.js mini web framework", // getting string
    host: "datajs.com", // host server
    port: 443, // port server,
    path: "/todos" // path server
});

queryURL() method in sphere.js

queryURL() the request from the client, as an object. 

const sphere = require("./sphere");
const app = sphere;
app.queryURL({
    head: 500,
    listen: 8000 
});

splitURL() method in sphere.js

There are built-in modules to easily split the query string into readable parts, such as the URL module. 

const sphere = require("./sphere");
const app = sphere;
app.splitURL({
    head: 500,
    listen: 8000 
});

URL() method in sphere.js

The URL method splits up a web address into readable parts. 

const sphere = require("./sphere");
const app = sphere;
app.URL("host",{
    link: "http://localhost:8080/default.htm?year=2017&month=february"
    // retruns 'localhost:8080'
});

const sphere = require("./sphere");
const app = sphere;
app.URL("pathname",{
    link: "http://localhost:8080/default.htm?year=2017&month=february"
    //returns '/default.htm'
});

const sphere = require("./sphere");
const app = sphere;
app.URL("search",{
    link: "http://localhost:8080/default.htm?year=2017&month=february"
    //returns '?year=2017&month=february'
});
